# CSS-in-JS: Why?

- Power of JS
- Extensible plugin architecture (_automation_, _vendor prefixing_)
- More powerful than inline styles
- Thinking in components (_style abstraction_)
- True scoped/namespaced selectors
- Cross concern code sharing (JS <-> CSS)
- Dead code elimination
- CSS unit tests

<Notes>

Inline styles

- don't support pseudo classes, media queries
- can't style `html`, `body` tags
  With JSS
- You get all this
- & SCSS like functionality (nested self reference selectors)

Abstraction

- @ component level, not document
- true rule isolation (plugin) - some CSS auto inherits from parents, event if not defined

CSS Modules

- hybrid approach
- Auto namespace styles based on file name
- access in js like object

</Notes>

- Learning curve
- New dependencies
- Harder to adopt for newer team members (more to learn)

## Object styling API

```js
import { css } from 'emotion';

const selector = css({
  'backgroundColor': '#0874da',
  'color': 'white',
  '&:hover': {
    backgroundColor: '#3482df',
    color: 'pink',
  },
});
```

---

## String styling API

```js
import { css } from 'emotion';

const selector = css`
  background-color: #0874da;
  color: white;

  &:hover {
    background-color: #3482df;
    color: pink;
  }
`;
```
